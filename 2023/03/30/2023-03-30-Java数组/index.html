数组

数组本身其实是一个对象

  • 数组本身是由一连串连续的内存地址，因为是对象，所以存储在堆中，每次初始化后新建于堆中。

数组内存储的都是相同的元素类型，且数量是固定的

  • 这里是因为有java的编译检查决定的，因此可以通过反射方式来使一个数组存储多种数据类型
  • 动态数组扩容的本质其实是新建一个数组，将原数组的数据拷贝过去，然后将新数组的地址赋给变量

数组的索引其实是从0开始的。

为什么？Java其实是基于c/c++实现的，在c中数组的索引就是从0开始。c中有个很重要的概念就是指针，指针本质上就是一个偏移量，表示相对于初始位置偏移了多少位置，第一个元素其实就是其本身，因此偏移量就是0，索引开始也是0

数组可以存储基本数据类型，也可以存储引用数据类型(自己创建的类对象)

String不可变

首先是String源码

  1. String是被final修饰的

    意味着String类无法被继承，子类也无法修改它的行为

  2. String 源码中存储值的char[] 数组也是被final修饰的，因此里面的值无法被修改

  3. String继承了Serializable接口，意味着它可以被序列化

  4. String实现了Compareble接口，意味着应该用compareTo来进行比较，但是我们一般都是用equals方法

为什么要这么设计？

  1. 保证String对象的安全性，因为一般我们的密码字段都是用的String来进行存储的

  2. 可以实现字符串常量池

    我们操作String对象生成的一般都是新的String对象

String 对象一旦被创建后就固定不变了，对 String 对象的任何修改都不会影响到原来的字符串对象，都会生成新的字符串对象

  1. String类由于不变的特性，适合用于作HashMap的key值，如果可以经常变更的化，hashmap性能会很差

字符串常量池

字符串常量池是由于在程序中，字符串创建和使用频率实在很高，且由于字符串不变的特性，为了节省内存和空间的消耗，因此设计出来专门存储内存中已创建过的字符串对象的。

也正是因为由字符串常量池的存在，我们在定义字符串的时候可以直接

在我们创建 “peace”这个字符串的时候，Java虚拟机会先在字符串常量池中查找这个对象，若有，那么我们就直接把常量池中的对象赋值给变量a。如果没有，那么我们就先在常量池中创建”peace”字段，然后再把这个对象赋值给变量a

这里又得提到另一种创建方式

这里和上面不同的是，如果字符串常量池中没有”peace”字符串的话，这里就会创建两个对象

首先Java虚拟机会先到字符串常量池中去寻找”peace”字段，如果没有就创建”peace”字段，如果有，就直接把该字段传给堆中，在堆上创建一个”peace”的字符串对象，然后再把值赋值给变量a。

意思是，无论字符串常量池有没有这个字符串，都会在堆中创建一个对象

这一点可以使用 String.intern() 方法 来减少对象的创建，直接返回在字符串常量池的引用

可以理解为字符串常量池是一个 “历史字符串记录器” 它会记录你所有关于字符串的常见操作，然后将这个对象拷贝一份放到自己的那一块空间当中，在我们通过 “ “ 方式创建字符串时，就可以不用再在堆上创建。直接从这个字符串常量池中取，可以提高性能

字符串常量池在 JDK 1.8之后就到了堆中

永久代和元空间的概念

  • 方法区是 Java 虚拟机规范中的一个概念，就像是一个接口吧；
  • 永久代是 HotSpot 虚拟机中对方法的一个实现，就像是接口的实现类；
  • Java 8 的时候，移除了永久代，取而代之的是元空间，是方法区的另外一个实现。

永久代是放在运行时数据区中的，所以它的大小受到 Java 虚拟机本身大小的限制，所以 Java 8 之前，会经常遇到 java.lang.OutOfMemoryError: PremGen Space 的异常，PremGen Space 就是方法区的意思；而元空间是直接放在内存中的，所以只受本机可用内存的限制，虽然也会发生内存溢出，但出现的几率相对之前就小了很多。

String.intern()方法

  • 直接使用双引号声明出来的String对象会直接存储在常量池中。

  • 如果不是用双引号声明的String对象，可以使用String提供的intern方法。intern 方法会从字符串常量池中查询当前字符串是否存在，如果常量池中存在当前字符串, 就会直接返回当前字符串，若不存在就会将当前字符串放入常量池中

在JDK 1.8 之后，这个”放入”操作会变成引用，即若不存在，就会将队中哪个String对象的地址放入字符串常量值中，以减少字符串移动的消耗

不过需要注意的是，尽管 intern 可以确保所有具有相同内容的字符串共享相同的内存空间，但也不要烂用 intern，因为任何的缓存池都是有大小限制的，不能无缘无故就占用了相对稀缺的缓存空间，导致其他字符串没有坑位可占。

另外，字符串常量池本质上是一个固定大小的 StringTable，如果放进去的字符串过多，就会造成严重的哈希冲突，从而导致链表变长，链表变长也就意味着字符串常量池的性能会大幅下降，因为要一个一个找是需要花费时间的。

分割字符串

分割字符串一般使用 String.split() 方法，分割前可以先检测字符串内是否包含分隔符，而且最好使用正则表达式来表示分隔符，否则容易引起错误。

正则表达式的断言模式还可以让我们灵活地选择将分隔符追加到哪一部分，

String.split() 也可以传两个参数，第二个参数可以传拆分的字符串个数。当字符串个数达到时，就不会继续拆分了

对象

1.方法

访问权限：它指定了方法的可见性。Java 提供了四种访问权限修饰符：

  • public：该方法可以被所有类访问。
  • private：该方法只能在定义它的类中访问。
  • protected：该方法可以被同一个包中的类，或者不同包中的子类访问。
  • default：该方法如果没有使用任何访问权限修饰符，Java 默认它使用 default 修饰符，该方法只能被同一个包中类可见。

三大特性

1.封装

java 的封装性意义是对外隐藏实现细节，对外部开发操作对象的方法。这样做有几个好处：

  1. 确保了数据的安全性

    Java对象的封装让我们并不能随意的修改对象中的数据，而需要我们遵循给定的接口或者方法来访问数据，而且对象也可以封装一些对外不可见的 属性 来进行必要的逻辑运算

  2. 便于我们对数据做出更加精细的处理

    这也可以称做数据合理性校验，因为我们必须使用对象提供的方法来操作数据(包括设置修改数据)，因此我们可以在方法中来加入一些入参的逻辑判断，若入参未按照要求，我们可以对该情况进行处理(抛出异常，个数转换)

  3. 便于我们统一进行对象修改

2.继承

继承是面向对象软件中的一个重要概念，它让我们复用之前的代码变得容易，可以极大提升我们开发的效率

继承就是子类继承父类的属性和方法，让子类对象具有父类非私有的 属性，方法 或者子类从父类继承方法，使得子类具有父类的 同种方法

这个同种并不是完全相同，因为子类可以重写父类具有的方法，从而使子类拥有父类不曾具有的特性

继承时各个方法的执行顺序

子类在新建时会默认执行父类的无参构造，这是编译器自己默认添加的，当然也可以自己手动覆盖。

大概顺序为:

  1. 父类中静态成员变量和静态代码块
  2. 子类中静态成员变量和静态代码块
  3. 父类中普通成员变量和代码块，父类的构造方法
  4. 子类中普通成员变量和代码块，子类的构造方法

总的来说，就是静态>非静态，父类>子类，非构造方法>构造方法。同一类别（例如普通变量和普通代码块）成员变量和代码块执行从前到后，需要注意逻辑。

继承(抽象类)和实现接口的区别

  1. 继承时我们可以让子类拥有和父类相同的属性和方法，利于我们复用代码，而接口只是提供一个方法的标准，需要我们手动去复写

  2. 继承只能支持单继承，而接口可以实现很多个

    实现多继承的方法：

    1. 继承可以是链式的，一个子类可以继承到父类的父类，子类又拥有所有父类公开的属性和方法

      但是Java中继承的层数不宜太多，因为类在新建的时候会先将父类新建，再去新建子类特有的属性，这叫做内存叠加，因此我们如果继承的层数太多，会新建很多无效对象, 增加了资源消耗

    2. 可以在类中新建个内部类，继承其他对象，也可以达到复用属性的目的，但是意义并不大

    3. 将其他继承类作为成员变量

  3. 继承是 is-a 的关系，而接口类似于 like-a 的关系

3.多态

多态是程序中定义的引用变量所指向的具体类型和通过该引用变量所发出的方法在编译时期并不确定，只有在程序运行期间才确定

即一个引用变量到底指向哪个实例变量，这个引用变量发出的调用到底又实现的是哪个类的什么方法，由程序在运行期间决定

因为在程序运行时才确定具体的类，这样，不用修改源程序代码，就可以让引用变量绑定到各种不同的类实现上，从而导致该引用调用的具体方法随之改变，即不修改程序代码就可以改变程序运行时所绑定的具体代码，让程序可以选择多个运行状态，这就是多态性。

也就是方法定义的时候可以传入 接口 或者 父类 ，在传参时可以传入任意实现了该接口或者类的对象，这样在编译时虽然编译的是接口，但是实际上在调用的过程中指向的是实现了该接口的对象，由此我们就可以在不更改源代码的情况下，通过改变入参来实现更改函数行为的操作，这就是多态

多态发生条件

  1. 发生继承
  2. 子类重写父类方法或者对属性重新赋值
  3. 父类引用指向子类对象

集合

1. ArrayList

ArrayList 实现了Cloneable (克隆),Serializable (序列化)，RandomAccess (快速访问) 接口

为什么ArrayList 内部数据部分用transient来修饰

序列化：

序列化是指将对象转化成字节序列的形式来表示，用于文件，网络传输，写入数据库等操作

使用transient修饰的变量不会被序列化——为什么？

由于ArrayList 动态扩容的特性，每次数组要满的时候都会扩容1.5倍，因此ArrayList 永远都会有空余的空间，这些空间在数据量大的时候，会变得更大，属于无效内容，可是序列化会将这些空的内存一起加入序列化，导致耗时增加很多(序列化耗时较长)，因此，ArrayList 自己提供了writeObject() 和 readObject()两个方法来进行数组的序列化，即每次序列化传入数组实际的数据，这样就大大减少了 ArrayList 序列化所需要消耗的时间

RandomAccess (快速访问) 接口

接口内部是空的，属于一种标记，标记这个类支持快速随机访问，即 可以不遍历而直接通过下标(索引访问)到内存地址
